Setting Up Your Environment

Jun He May 31, 2026

Empirical finance work is only as trustworthy as the environment it runs in. Before touching data, it pays to fix a reproducible setup so that the same script produces the same numbers on any machine, today or a year from now.

Language and packages

Pick one ecosystem — R (with the tidyverse) or Python (with pandas/numpy) — and install a small, stable set of packages: data manipulation, plotting, a database driver, and a statistics/econometrics library. Pin versions where you can, so an upgrade upstream does not silently change a result.

Project layout

Keep a project self-contained: a folder per project, scripts separated from raw data, and generated outputs (figures, tables, cleaned datasets) written to their own directory rather than mixed in with code. Relative paths from the project root keep everything portable.

Credentials

Several later steps pull from authenticated sources such as WRDS. Store credentials outside the codebase — environment variables or a local config file that is never committed — so scripts can authenticate without ever hard-coding a password into version control.

Reproducibility

Set a random seed wherever you draw random numbers, record your package versions, and prefer scripted downloads over manual ones. The goal is that a reader could clone the project and regenerate every figure end to end.
